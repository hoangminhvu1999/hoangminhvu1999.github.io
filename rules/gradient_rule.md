# Gradient Color Rules for List Items

This document defines a systematic gradient color rule for all list items across the application. Each index position in any list (Projects, Experience, Skills, etc.) will use the same gradient color, ensuring visual consistency throughout the portfolio.

## Implementation Guide

### How to Apply

For any list in your application, apply the gradient based on the item's position:

```html
<!-- Example: Experience List -->
<!-- Item 1 uses Index 1 gradient -->
<div class="bg-gradient-to-r from-indigo-500 to-purple-600">
  Experience #1
</div>

<!-- Item 2 uses Index 2 gradient -->
<div class="bg-gradient-to-r from-purple-500 to-pink-600">
  Experience #2
</div>

<!-- Item 3 uses Index 3 gradient -->
<div class="bg-gradient-to-r from-green-500 to-emerald-600">
  Experience #3
</div>
```

### Cycling Rule

If you have more than 25 items in a list, cycle back to Index 1:
- Item 26 → Use Index 1 gradient
- Item 27 → Use Index 2 gradient
- And so on...

### Consistency Across Lists

**Example Application:**
- **Projects List:** Project #1 (Indigo-Purple), Project #2 (Purple-Pink), Project #3 (Green-Emerald)
- **Experience List:** Experience #1 (Indigo-Purple), Experience #2 (Purple-Pink), Experience #3 (Green-Emerald)
- **Blog Posts:** Post #1 (Indigo-Purple), Post #2 (Purple-Pink), Post #3 (Green-Emerald)

This creates a **unified visual language** across your entire application.

---

## Gradient Direction Variations

While the default is `to-r` (left to right), you can adjust the direction based on the component:

- **Headers:** `bg-gradient-to-r` (horizontal)
- **Cards:** `bg-gradient-to-br` (diagonal)
- **Backgrounds:** `bg-gradient-to-b` (vertical)

**Example:**
```html
<!-- Card header with diagonal gradient -->
<div class="bg-gradient-to-br from-indigo-500 to-purple-600">
  Card Header
</div>
```
